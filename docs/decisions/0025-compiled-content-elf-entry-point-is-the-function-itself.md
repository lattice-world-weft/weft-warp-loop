# Compiled-content ELFs are freestanding, with entry point set directly to the compiled function

## Status

Accepted; implemented (`flow-toolchain/examples/ir_compiled_content_test.cpp`,
checkpoint 4 of the ADR 0013/0014 compiler).

## Decision

`s7_guest.elf` ([ADR 0006](0006-libriscv-sandboxed-s7-lisp-over-native-janet.md))
needs a real newlib `_start`/`main` to run `s7_init()` before any
`vmcall` is useful — its content is dynamic Scheme text evaluated at
call time. A compiler-emitted function ([ADR 0024](0024-emit-riscv-assembly-text-not-hand-encoded-machine-code.md))
needs none of that: it's already pure compiled code with no
initialization step. Linked freestanding (`-static -nostdlib
-nostartfiles`), with the ELF's own entry point set directly to the
compiled function via `-Wl,-e,<symbol>` — avoiding a `_start` stub
entirely, since `vmcall()` never executes from `e_entry` (it jumps to
the named symbol directly); the linker still requires *some* valid
entry to emit an `EXEC`, not a relocatable object, so pointing it at the
one real function costs nothing and needs no stub code.

Verified: two independent `Machine<RISCV64>` instances, given the
identical `vmcall<2'000'000ull>("Fanoutcore_hysteresisTicksFor", 6)`,
both returned `3` (matching `max 1 (6/2)`) in exactly `8` instructions
each — the same determinism bar ADR 0006 already established for the
interpreted s7 guest, now proven for compiled content too. Following
`s7_guest.elf`'s own precedent, the linked `.elf` is a committed
artifact, not a CMake build output (the RISC-V toolchain isn't a build
dependency of this project); its `.s`/`.o` intermediates are gitignored
and regenerated by every run of `IrCodegenScratch.lean`.

## Consequences

Good: compiled content needs no shared runtime/init step at all — each
compiled declaration is its own tiny, self-contained, freestanding ELF,
simpler than the interpreted guest's model. The determinism proof reused
ADR 0006's exact methodology rather than inventing a new one. Bad: this
test still calls `setup_linux()`/`setup_linux_syscalls()` (the full
Linux syscall surface, same as `s7_guest.elf`) even though the compiled
function makes zero syscalls — the default-deny narrowing ADR 0006
already flagged as unstarted stays unstarted here too, now for a second
ELF shape as well as the interpreted one.
